# Communication rules

## Formatting

- No emoji in code, commit messages, comments, or documentation
- No decorative formatting — no ASCII art, no box-drawing characters in output
- Use plain markdown for documentation
- Keep responses concise — skip explanations for obvious things

## Commit messages

- Conventional commit format: type(scope): description
- Types: feat, fix, refactor, docs, test, chore, perf, ci
- Imperative mood: "add feature" not "added feature"
- Max 72 characters for subject line
- Body explains why, not what

## Code comments

- Comments explain WHY, not WHAT — the code shows what
- No commented-out code — delete it, git has history
- No TODO comments without a linked issue or task
- Use the language's standard doc format (JSDoc, docstring, godoc)

## File naming

- README, CHANGELOG, LICENSE: uppercase (convention)
- Everything else: lowercase with hyphens or underscores per language convention
- No spaces in filenames ever
